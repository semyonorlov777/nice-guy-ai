Все заметные изменения платформы Nice Guy AI.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [Unreleased]

### Добавлено
- **Тест «Какой у тебя движок развития?» для «100 записок о развитии» (Пётр Осипов)** — 25 вопросов, 5 шкал (работа со страхом, масштаб мышления, архитектура энергии, гигиена окружения, самонаблюдение). Навыковый формат (`higher_is_better`), уровни Новичок→Развивающийся→Уверенный→Мастер. Центральная метарамка из Осипова: «развитие через удовольствие, не через преодоление»
- **5 тем для «100 записок о развитии»** с привязкой к шкалам теста (Работа со страхом, Масштаб мышления, Архитектура энергии, Гигиена окружения, Самонаблюдение) — персонализируют хаб по результатам
- AI-приветствие на хабе «100 записок о развитии» (3 состояния: первый визит / вернулся с тестом / вернулся без теста)
- 5 иконок тем 100-notes (fear_mastery, scale_thinking, energy_agency, environment_hygiene, self_reflection) в `THEME_ICON_MAP` — переиспользованы существующие SVG (Shield, Rocket, InstrumentLightning, UsersLove, BookOpen)
- Документация Deep Research процесса в `docs/books/`: оба промпта + сырые результаты + финальная инвентаризация. Закрывает прецедент книги без полного текста — 4 HIGH цитаты по окружению получены из интервью Петра Осипова в MC.today от 18.10.2019 с явной дисамбигуацией от Алексея Осипова («Будни муз») и Алексея Ильича Осипова (богослов)
- **5-й режим «Наведение открытости» (hypno_trance) для «Разговорного гипноза»** — тренажёр по живой речи и силе паузы (гл. 1 Бакирова). 5 фаз: контекст → разбор «живое vs абстрактное» → где ставить паузу → темп и ритм → итог. Закрывает пятую шкалу теста (`hypno_trance_access`) — теперь у каждой шкалы есть свой полноценный режим
- **Тест «Какой ты собеседник?» для программы «Разговорный гипноз» (Бакиров)** — 25 вопросов, 5 шкал (раппорт, мастерство внушений, выбор стратегии, перехват инициативы, наведение открытости). Первый **навыковый** тест платформы (`higher_is_better`, уровни Новичок→Развивающийся→Уверенный→Мастер) — карта навыков, не диагностика паттернов
- **5 тем для «Разговорного гипноза»** с привязкой к шкалам теста (Раппорт и роль, Мастерство внушений, Выбор стратегии, Перехват инициативы, Наведение открытости) — позволяют персонализировать хаб по результатам теста
- AI-приветствие на хабе «Разговорного гипноза» (3 состояния: первый визит / вернулся с тестом / вернулся без теста)
- 5 новых иконок тем (rapport, suggestions, strategy, awareness, trance) в `THEME_ICON_MAP`
- Система документации проекта: ADR, runbooks, схема БД, справочник env
- Карточка теста на хабе программы Games People Play («Какие игры ты играешь?») — тест существовал в БД, но был невидим без mode_template/program_mode
- System prompt для GPP-теста (`programs.test_system_prompt`) — улучшает качество интерпретации текстовых ответов в тесте
- Quick-reply кнопки в режимах «Славный парень или нет?» и «Теория книги» (4 кнопки в каждом)
- Аудиты обеих программ с тестами (`.claude/skills/book-audit/findings/`)
- AI-приветствие на хабе программы Games People Play (3 варианта: первый визит / вернулся с тестом / вернулся без теста) — раньше на месте текста был пустой золотой кружок
- 5 тем для Games People Play с привязкой к шкалам теста (Твои игры, Родитель-Взрослый-Ребёнок, Жизненный сценарий, Голод по признанию, Треугольник Карпмана) — позволяют персонализировать хаб по результатам теста
- Дебаг-параметр `?test_state=welcome|history-single|history-multi` для визуального тестирования welcome-экрана теста без данных в БД

### Изменено
- **Модель Gemini для чата и тестов вынесена в env-переменную `GEMINI_CHAT_MODEL`** (default `gemini-2.5-flash`, рекомендуется `gemini-3-flash-preview`). Хелпер `chatModel()` + `CHAT_PROVIDER_OPTIONS` в `lib/ai.ts`, `thinkingConfig.thinkingBudget: 0` для экономии output-токенов. Переключение — через Vercel env без деплоя; откат = удалить переменную
- `appendTestScores()` теперь сам инжектит правило «не зачитывай числовые баллы вслух» — раньше дублировалось в каждом промпте
- ta_diagnostic в Games People Play теперь корректно открывается с хаба (раньше вёл в свободный чат)
- Хаб больше не использует хардкод-таблицу tool keys — derive из mode.key, новый chat-based режим автоматически роутится без правок UI
- Удалён disabled режим `self_work` из nice-guy, sort_order перенумерован 0–9 без дублей
- Чеклист `book-audit` v1.5: добавлены проверки `programs.hub_messages` (E13) и динамического заголовка `HistoryScreen` (E14); новая секция I про паттерн `?<screen>_state=` для дебага UI-состояний

### Исправлено
- **`RadarChart` и `TestResultsPage` стали data-driven по `score_direction`** — раньше цвета (`dotColor`, лейблы зон ВЫСОКИЙ/СРЕДНИЙ/НИЗКИЙ) и fallback-уровни (`getLevelLabel`) были захардкожены под `lower_is_better`. Для навыкового теста (Бакиров) высокий балл (мастерство) корректно отображается зелёным, низкий (зона роста) — красным. ISSP/GPP остались как раньше (default `lower_is_better`)
- Экран истории теста больше не показывает «Индекс Синдрома Славного Парня» для программ, отличных от nice-guy — заголовок и бейдж берутся из `testConfig.ui_config.welcome_title/welcome_badge`
- Карточка теста в секции «Инструменты» хаба теперь **всегда зелёная** — ранее цвет зависел от `program_modes.color_class` в БД (у nice-guy был green, у GPP — accent/золотой), из-за чего тест GPP визуально сливался с чат-режимами. Правило применяется автоматически для любого test-режима (`is_chat_based=false` + `route_suffix` начинается с `/test/`)
- `HistoryScreen` теста больше не прижат к верху у авторизованных пользователей с предыдущими результатами — добавлен `justify-content: center` в `.tc-history-screen`, контент отцентрован по вертикали как в `WelcomeScreen`/`BlockTransition`/`CompletionScreen`

# Голосовой ввод — Как это работает

---

## Исходное состояние

Поле ввода пустое. Справа — **кнопка микрофона** 🎤 (вместо кнопки отправки ↑).
Как только пользователь начинает печатать текст — микрофон заменяется на кнопку отправки ↑.
Как только текст стёрт — снова микрофон.

---

## Сценарий 1: Короткое голосовое (push-to-talk)

Пользователь хочет быстро ответить голосом.

1. **Зажимает палец** на кнопке 🎤
2. Кнопка меняет вид — становится красной, пульсирует
3. Появляется панель записи: красная точка + таймер (0:01, 0:02...)
4. Текст распознаётся **в реальном времени** и показывается в панели записи (серым курсивом)
5. **Отпускает палец**
6. Кратковременно показывается "Распознаю..." (0.3-1 сек)
7. **Сообщение автоматически отправляется в чат** — как будто пользователь набрал текст и нажал Enter
8. AI начинает отвечать

**Итого:** зажал → сказал → отпустил → улетело. Ноль лишних действий.

---

## Сценарий 2: Длинное голосовое (lock mode)

Пользователь хочет выговориться — 1, 3, 10 минут. Держать палец всё это время невозможно.

1. **Зажимает палец** на кнопке 🎤
2. Начинается запись (как в Сценарии 1)
3. **Не отпуская палец, ведёт его вверх** — над кнопкой появляется иконка замочка 🔒
4. Палец доходит до замочка → **замочек "защёлкивается"** (анимация)
5. **Отпускает палец** — запись **продолжается**! Руки свободны.
6. Панель записи показывает: таймер + текст в реальном времени
7. Кнопка справа теперь — **Send ↑** (не стоп)
8. Пользователь говорит сколько хочет
9. **Нажимает Send ↑**
10. Кратковременно "Распознаю..."
11. **Сообщение отправляется в чат**

**Итого:** зажал → свайп вверх → отпустил → говоришь свободно → Send → улетело.

---

## Сценарий 3: Отмена записи

Пользователь передумал.

### Вариант А: Свайп влево
1. Во время записи (палец зажат) — **ведёт палец влево**
2. Появляется красная зона / текст "Отменить"
3. Палец доходит до зоны отмены → **запись отменена**
4. Всё возвращается в исходное состояние
5. Ничего не отправлено

### Вариант Б: Кнопка ✕
1. В lock mode (руки свободны) — слева в панели записи есть кнопка **✕**
2. Нажимает ✕ → **запись отменена**
3. Всё возвращается в исходное состояние

---

## Сценарий 4: Микрофон недоступен

Браузер не поддерживает Web Speech API (Firefox, Safari) или пользователь запретил микрофон.

### Если нет Web Speech API и это авторизованный чат:
- Используется **платный fallback** (MediaRecorder + OpenAI)
- Всё работает так же как Сценарии 1-3
- Единственная разница: нет реалтайм-текста во время записи (вместо него — waveform визуализация)
- После остановки: задержка 1-3 сек на распознавание (сервер)
- Под панелью записи мелкий текст: "платное распознавание · ~50 токенов/мин"

### Если нет Web Speech API и это анонимный чат:
- Кнопка микрофона **не показывается**
- Только текстовый ввод

### Если пользователь запретил микрофон:
- При нажатии на 🎤 — тултип: "Разрешите доступ к микрофону"
- Кнопка остаётся, но не работает

---

## Лимиты

| Контекст | Максимум записи |
|---|---|
| Анонимный чат (лендинг) | 5 минут |
| Авторизованный чат | 60 минут |

- За 30 секунд до лимита: таймер становится **оранжевым**
- При достижении лимита: автостоп → распознавание → отправка

## Визуальные состояния панели ввода

### Обычное (не записывает):
```
[ Или напиши своими словами...    ] [ 🎤 ]     ← input пуст
[ Привет, хочу рассказать...      ] [ ↑  ]     ← есть текст
```

### Запись (палец зажат):
```
[ 🔴 0:03  "и я хочу рассказ..."  ] [ 🎤 ]     ← пульсирует красным
                                       🔒        ← замочек над кнопкой
```

### Lock mode (руки свободны):
```
[ ✕  🔴 1:42  "думаю что это..."  ] [ ↑  ]     ← кнопка Send
```

### Распознавание (после остановки):
```
[ ⏳ Распознаю...                  ] [ ⏳ ]     ← 0.3-3 сек
```

## Защита от потери сообщений

Самое страшное — человек говорил минуту, а сообщение пропало. Ниже все сценарии и как от них защититься.

### 1. Интернет пропал во время записи

Web Speech API отправляет аудио на серверы Google. Нет сети = ошибка.

**Защита:** Если Web Speech вернул ошибку "network" — **не выбрасывать то что уже распознано**. Показать пользователю: "Связь потеряна. Вот что удалось распознать:" и вставить накопленный текст в textarea. Пользователь сможет отправить вручную когда сеть вернётся.

### 2. Chrome обрывает распознавание при паузе

Замолчал на 5-10 секунд — Chrome решил что всё, вызвал onend.

**Защита:** Auto-restart. При onend — если запись ещё активна (state = recording или locked), автоматически перезапускать recognition. **Никогда не обнулять накопленный текст** при перезапуске. Пользователь не должен замечать перезапуски.

### 3. Вкладка ушла в фон (переключился в другое приложение)

На мобиле: пришло уведомление, ткнул — браузер убивает запись.

**Защита:** Слушать событие `visibilitychange`. Если вкладка ушла в фон и запись шла — **автостоп + отправить что есть**. Не ждать возврата пользователя. Лучше отправить неполное чем потерять всё.

### 4. Экран телефона погас

То же что п.3 — экран погас = вкладка в фон.

**Защита:** Та же — `visibilitychange` → автостоп → отправить накопленное.

### 5. Случайная отмена (свайп, ✕)

Палец дёрнулся влево или случайно ткнул ✕.

**Защита:** Если записано **больше 5 секунд** — показать подтверждение: "Отменить запись? (записано 0:47)". Две кнопки: "Да, отменить" и "Нет, продолжить". Для коротких записей (<5 сек) — отменять без подтверждения.

### 6. Распознавание вернуло пустой текст

Web Speech распознал тишину или Google вернул пустоту.

**Защита:** **Не отправлять пустое сообщение.** Показать: "Не удалось распознать речь. Попробуйте ещё раз." Вернуть в исходное состояние.

### 7. Ошибка отправки в чат

Текст распознан, но handleSend упал — нет chatId, сервер 500, сеть пропала.

**Защита:** Если отправка провалилась — **вставить текст в textarea как fallback**. Текст не потерян, пользователь видит его и может отправить повторно. Показать сообщение: "Не удалось отправить. Текст сохранён в поле ввода."

### 8. Платный fallback: таймаут OpenAI

Длинная запись (5+ мин), большой файл, OpenAI долго обрабатывает или таймаутит.

**Защита:** Retry 1 раз автоматически. Если повторно ошибка — показать: "Ошибка распознавания. Повторить?". Blob сохранён в памяти, можно повторить без перезаписи.

### 9. Платный fallback: баланс кончился

Начал запись, баланс был, а к моменту отправки — уже ноль (потратил в другой вкладке).

**Защита:** Проверять баланс **ДО начала записи**. Если баланс < минимума — показать: "Недостаточно токенов для голосового ввода. Пополните баланс." Кнопка микрофона неактивна.

> **Заметка на будущее:** При реализации системы уведомлений о балансе — не забыть про этот сценарий. Предупреждать когда баланс приближается к нулю, в т.ч. в контексте голосового ввода.

---

## Что НЕ нужно

- ❌ Вставлять текст в textarea для редактирования
- ❌ Ждать что пользователь нажмёт Enter после записи
- ❌ Показывать текст и спрашивать "отправить?"
- ❌ Любые дополнительные шаги между "стоп" и "отправлено"

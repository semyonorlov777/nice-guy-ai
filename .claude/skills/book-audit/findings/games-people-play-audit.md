## Рекомендации

### Приоритет 1 (критично)

#### ⚡ ta_diagnostic недоступен с хаба

**Статус:** ❌
**Проблема:** `toolKeyMap` в `components/hub/InstrumentList.tsx:82-107` не содержит `ta_diagnostic`. Главная free-карточка GPP (sort_order=0) генерирует href `/program/games-people-play/chat` (просто свободный чат), а не `/chat/new?tool=ta-diagnostic`. Пользователь, кликая на «Диагностика ТА», попадает в свободный чат с system_prompt `free_chat`.
**Решение:**
```ts
// components/hub/InstrumentList.tsx:82
const toolKeyMap: Record<string, string> = {
  // ...
  ta_diagnostic: "ta-diagnostic",  // ← добавить
  ta_game_quiz: "ta-game-quiz",
  // ...
};
```
Параллельно — отказаться от хардкод-мапа: можно вычислять `ta_diagnostic → ta-diagnostic` через `mode.key.replace(/_/g, "-")` (идея: hyphen→underscore работает в обе стороны через `lib/queries/welcome.ts:45`). Хардкод-мап `toolKeyMap` — антипаттерн (нужно помнить добавлять при каждом новом режиме).
**Трудозатраты:** 5 мин (добавить запись) или 30 мин (рефакторинг)
**Приоритет:** 1

#### E9/E10 — Нет mode_template + program_mode для теста gpp-test

**Статус:** ❌
**Проблема:** В БД нет `mode_template` с `route_suffix LIKE '/test/%'` для GPP. Согласно PLATFORM_MAP «Без этого нет карточки на хабе» — пользователь не увидит тест в InstrumentList. Войти в тест можно только напрямую по `/program/games-people-play/test/gpp-test`.
**Решение:** Выполнить блок 5 из шаблона PLATFORM_MAP:
```sql
INSERT INTO mode_templates (key, name, description, icon, chat_type, route_suffix, is_chat_based)
VALUES ('test_gpp', 'Какие игры ты играешь?', 'Тест на выявление психологических игр', 'check', NULL, '/test/gpp-test', false)
ON CONFLICT (key) DO NOTHING;

INSERT INTO program_modes (program_id, mode_template_id, sort_order, enabled, access_type)
SELECT (SELECT id FROM programs WHERE slug='games-people-play'),
       (SELECT id FROM mode_templates WHERE key='test_gpp'),
       0, true, 'free'
ON CONFLICT DO NOTHING;
```
sort_order=0 (тест первый). Существующий ta_diagnostic перенумеровать на 1, остальные сдвинуть.
**Трудозатраты:** 15 мин
**Приоритет:** 1

#### E6 — programs.test_system_prompt = NULL

**Статус:** ❌
**Проблема:** `select test_system_prompt FROM programs WHERE slug='games-people-play'` возвращает NULL. PLATFORM_MAP писал «404», но фактический код (`app/api/test/route.ts:189`) делает `?? ""`. Поведение: AI текстовых ответов теста получает пустой system prompt → ответы низкого качества (не понимает контекст ТА).
**Решение:** Добавить `test_system_prompt` для GPP:
```sql
UPDATE programs
SET test_system_prompt = E'Ты помогаешь пользователю проходить тест по транзактному анализу Эрика Берна...'
WHERE slug = 'games-people-play';
```
Скопировать структуру из nice-guy `test_system_prompt` (19356 символов), адаптировать под ТА-терминологию.
**Трудозатраты:** 30 мин
**Приоритет:** 1

#### E3/E4 — TA-промпты не используют test scores и не запрещают их зачитывать

**Статус:** ❌
**Проблема:** `appendTestScores()` инжектит JSON `scores_by_scale` во все чаты для пройденного теста. Но **ни один TA-промпт не содержит** ни инструкции «используй scores для персонализации», ни запрета «не зачитывай числа». Эффект: тест пройден, но режимы его «не видят». Хуже того — при инжекте JSON AI может зачитать «у тебя 4.2 по ta_game_tendency», что разрушает терапевтический контекст.
**Решение (одно из двух):**
- (A) Добавить в каждый TA-промпт секцию «КРОСС-РЕЖИМНАЯ СВЯЗКА → если есть scores_by_scale» (как в nice-guy ng_my_syndrome) — 8 правок
- (B) Лучше: вынести правило «не зачитывай числа, используй для навигации» в `appendTestScores()` (см. рекомендацию для nice-guy E4) — 1 правка
**Трудозатраты:** 30 мин (B) или 1 час (A)
**Приоритет:** 1

### Приоритет 2 (улучшение)

#### B4/B5 — welcome_replies хранятся как строки, не объекты

**Статус:** ⚠️
**Проблема:** Все 8 TA-режимов имеют `welcome_replies` как `["текст1", "текст2"]`. PLATFORM_MAP строго требует `[{text, type}]`. `normalizeWelcomeReplies` нормализует в runtime, но все reply становятся `type:"normal"` — теряется визуальное отличие безопасного выхода.
**Решение:** UPDATE для всех 8 TA-режимов: переписать welcome_replies в формат объектов, последний — `{text:"Мне сложно сформулировать", type:"exit"}`.
**Трудозатраты:** 15 мин (один SQL)
**Приоритет:** 2

#### A2 — author_chat не имеет system_prompt

**Статус:** ⚠️

**Решение:** Перенести содержимое из `programs.author_chat_system_prompt` в `program_modes.system_prompt` для единообразия с nice-guy. Иначе — задокументировать исключение.

**Приоритет:** 2

#### H5 — ta_game_quiz без metacognitive calibration

**Статус:** ⚠️
**Проблема:** Чеклист H5 требует «metacognitive calibration» в экзаменационных режимах. ta_game_quiz имеет адаптивную сложность («Кейсы нарастающей сложности»), но не требует от пользователя оценить уверенность («1-10»). Это снижает качество калибровки знаний.
**Решение:** Добавить в Фазу 2 ta_game_quiz инструкцию: «После каждого кейса спроси: \"Насколько уверен в ответе? 1-10\". Если >7 — усложняй следующий кейс».
**Трудозатраты:** 10 мин
**Приоритет:** 2

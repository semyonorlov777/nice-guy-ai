# Book-to-Modes: Справочник принципов и шаблонов

## 1. Пять платформенных архетипов

Подтверждены на 3 книгах разных жанров. Каждая книга = комбинация из этих 5.

| # | Архетип | Суть | Блум | Варианты |
|---|---------|------|------|----------|
| 1 | **Лекция** | AI передаёт знания, пользователь усваивает | Запоминание, Понимание | Прямая, Сократовская, Комбинаторика (продвинутая) |
| 2 | **Анализ** | Пользователь приносит материал → AI разбирает через книгу | Анализ | Себя, Другого, Отношений, Эмоций, Текста/файла |
| 3 | **Воркшоп** | AI помогает создать артефакт (формулировку, план, скрипт) | Применение | Внешний артефакт (бизнес), Внутренняя работа (терапия) |
| 4 | **Ролевая игра** | AI в роли — пользователь практикует навык | Создание | Короткий setup (бизнес), Длинный setup + безопасный фрейм (терапия) |
| 5 | **Экзамен** | AI даёт кейс → пользователь решает → обратная связь | Оценка | Адаптивная сложность, метафорические уровни |

### Паттерн размножения

| Тип книги | Какой архетип «раздваивается» | Почему | Итого режимов |
|-----------|-------------------------------|--------|---------------|
| Бизнес / переговоры | Лекция ×3 | Много теории, мало внутренней работы | 7 + чат |
| Терапия / отношения | Анализ ×3 | Много граней внутренней работы | 7 + чат |
| Практика / навыки | Лекция ×2 + Анализ ×2 | Баланс теории и самоанализа | 7 + чат |
| Мотивация / мышление | Воркшоп ×2 + Анализ ×2 | Нужны артефакты + рефлексия | 7 + чат |

**Формула:** 5 базовых + 1-2 подвида самого релевантного = 6-7 режимов. Плюс **Свободный чат** (обязательный для каждой книги).

**Свободный чат** — catch-all для тех, кто пришёл с конкретным вопросом и не хочет выбирать режим. Обязателен для каждой книги.

### Реестр книг (обновлять после каждой новой)

| Книга | Тип | Объединение | Режимов | Персона | Решения |
|-------|-----|-------------|---------|---------|---------|
| Гловер «Славные парни» | Терапия | Нет (1 книга) | 7 + чат + автор | Бережный наставник | 3 анализа (себя/отношения/родители), ролевая (границы), экзамен (кейсы), сократовская лекция, exercises, ISSP тест → режимы маппинг, портрет |
| Берн «Игры» + «Люди» | Терапия-отношения | Да (2→1) | 8 + чат + автор | Внимательный аналитик | 3 части (Увидеть/Понять/Действовать), диагностика ТА как тест |
| Чепмен «5 языков любви» | Терапия-отношения | Нет (1 книга) | 2 + чат + автор | Внимательный аналитик | Минимум режимов, фокус на бесплатный чат + автор |
| Бакиров «Разговорный гипноз» | Практика/НЛП | Нет (1 книга) | 4 + чат + автор | Мастер-практик | Подстройка, переговоры, распознавание манипуляций, мастер внушений. Широкая ЦА |

Платформа gender-neutral. Per-book accent colors меняются, base остаётся.

### Рекомендация: «родители > один родитель»

В терапевтических книгах отец и мать — часто две стороны одного уравнения. Если книга фокусируется на одном родителе — проверь: нет ли скрытой роли второго?
- Отец ушёл → мать заполнила вакуум
- Мать критична → отец не защитил
- Один идеализирован → другой обесценен

Если оба родителя значимы — называй режим «Мои родители и я», а не «Мой отец/мать и я». Фазы: отец → мать → связка → артефакт.

## 4. Типология suggested replies

| Тип | Функция | Когда использовать |
|-----|---------|-------------------|
| **Вопрос к AI** | Запросить информацию | Лекция, теория |
| **Голос пользователя** | Выразить чувство/страх | Самодиагностика, анализ |
| **Голос сопротивления** | Озвучить типичный блок | Терапевтические режимы |
| **Навигация setup** | Двигать подготовку | Ролевая игра |
| **Меню упражнений** | Выбрать тему/сценарий | Welcome любого режима |
| **Sentence completion** | Продолжить фразу AI | Диагностика убеждений |
| **Калибровка реакции** | Отметить что зацепило | После большого блока |
| **Безопасный exit** | «Я не понял» / «Мне сложно» | Везде (обязательный) |

### Антипаттерны replies (НИКОГДА):
- Replies дублируют текст AI (баг конкурента)
- Replies смешивают роли — вопросы AI и ответы user в одном списке
- Replies от третьего лица или в форме вопроса AI
- Replies не адаптируются по глубине (постоянно 4 штуки)

### Scaffolding fading:
- Turns 1-5: 3-4 replies (снижение стартовой тревоги)
- Turns 6-12: 2 replies + акцент на поле ввода
- Turns 13+: 0 replies или 1 fallback «Мне сложно сформулировать»
- Метрика: если >70% ответов через клики → уменьшать быстрее

## 5. Промпт-шаблон (вставлять в каждый системный промпт)

```
## ПРАВИЛА ОТВЕТА (обязательные)

### Длина
- Максимум 2 абзаца, 60-80 слов за ответ
- Один вопрос за раз. НИКОГДА не задавай 2-3 вопроса в одном сообщении
- Если нужно больше: «Хочешь, расскажу подробнее?»

### Обратная связь
- НИКОГДА: «молодец», «отлично», «умница» (похвала личности)
- ВСЕГДА: хвали стратегию — «Ты заметил X — точное наблюдение, потому что Y»
- При ошибке: не давай ответ. Дай наводящий вопрос

### Suggested replies
- Replies = ТОЛЬКО реплики от первого лица пользователя
- НИКОГДА не дублируй в replies то, что уже спросил в тексте
- Turns 1-5: 3-4 replies
- Turns 6-12: 2 replies
- Turns 13+: 0 replies или 1 fallback «Мне сложно сформулировать»

### Безопасность
- Презумпция нормы: если поведение адекватно — скажи «Твоя реакция нормальна»
- Не ищи травмы где их нет
- Перед глубокой темой: «Опиши одним словом, что ты сейчас чувствуешь»

### Контекст
- Replies без предположений: «Сложность с близким человеком» (не «с мамой»)
- После уточнения — используй конкретику
- При возврате: «Что помнишь из прошлого разговора?» (не пересказывай сам)

### Ролевая игра
- Setup: максимум 2-3 обмена (имя + роль, что тренируем)
- В симуляции: ТОЛЬКО голос персонажа
- Обратная связь: отдельно, с маркером «💡 Совет»
- После игры: дебрифинг (чувства → удачный момент → зона роста)

### Активация опыта
- Перед каждой новой темой: «Вспомни ситуацию из жизни, когда...»
- Теория = поверх личного опыта
```

## 7. Антипаттерны из Design Bible (НИКОГДА)

### В интерфейсе
- Геймификация ради геймификации (стрики, бейджи, лидерборды)
- Модальные окна в момент уязвимости
- Публичный шеринг личного прогресса
- «Вы ответили неправильно» — в терапии нет неправильных ответов
- Уведомления с давлением («Ты пропустил 3 дня!»)
- Confetti / celebration за упражнение
- Skeleton screens в чате

### В промптах
- Стены текста (150-400 слов за ответ)
- Несколько вопросов за раз
- Похвала личности («молодец», «умница»)
- Overdiagnosis — любая ситуация = патология
- Replies-дубликаты текста AI
- Replies смешанных ролей
- Setup ролевой игры >3 обменов
- AI одновременно играет роль И даёт обратную связь (разделять визуально)
- AI не предлагает перейти к артефакту (пользователь не знает когда «хватит»)
- AI всегда находит проблему (overfit на поиск патологии)
- Банальные абстрактные советы без конкретики

### В тоне
- Давление, таймеры, чувство вины за пропуск
- Клинический терапевтический жаргон
- Менторский тон «сверху вниз»
- Предположения о поле/ориентации/семейном положении

## 8. Welcome-шаблон (единый для всех книг)

```
1. ЭМОДЗИ + НАЗВАНИЕ РЕЖИМА
2. ХУК: 1-2 предложения — продающий текст (результат, не процесс)
3. КАК ЭТО РАБОТАЕТ: описание формата (1-2 предложения)
4. ЧТО ВЫ ПОЛУЧИТЕ: 3 bullet points (конкретные результаты)
5. SUGGESTED REPLIES: 3-4 стартовых варианта
   - От первого лица (голос пользователя)
   - Конкретные, не абстрактные
   - Последний = безопасный exit: «Мне сложно сформулировать»
```

### Правила welcome replies:
- Replies = конкретные люди/ситуации, не абстракции (П34)
- Replies = меню упражнений (П41)
- Один reply может быть подсвечен как персонализированный (П21)

## 9. Кросс-режимная связка (архитектура)

Каждый режим может **передавать** и **получать** данные.

### Стандартные имена переменных

Используй эти имена при проектировании связок. Группы:

**Профиль пользователя (из диагностики/теста):**
- `life_position` — жизненная позиция / основной профиль
- `main_pattern` — главный паттерн (игра, язык, синдром — зависит от книги)
- `pattern_level` — уровень выраженности паттерна (low/medium/high)
- `scores_by_scale` — баллы по шкалам (JSON)

**Результаты анализа:**
- `user_[attribute]` — результаты анализа себя (язык, паттерн, стиль, проблема)
- `partner_[attribute]` — результаты анализа другого
- `identified_[pattern]` — выявленный паттерн/убеждение
- `dominant_style` — доминирующий стиль/состояние

**Результаты воркшопа/ролевой:**
- `action_plan` — созданный план действий
- `practiced_scenario` — что тренировали в ролевой
- `successful_phrases` — фразы, которые сработали
- `permission_statement` — формулировка разрешения/нового убеждения

**Мета:**
- `resistance_points` — на что сопротивлялся
- `growth_areas` — зоны роста
- `theory_covered` — какие концепции уже знает

### Формат записи связок

При описании кросс-режимных связок используй формат:
```
ПЕРЕДАЁТ → «Название режима»: variable_name (тип: string|number|json)
ПОЛУЧАЕТ ← «Название режима»: variable_name (тип)
```

### Как инжектить:
В системный промпт добавляется переменная `{{cross_mode_data}}` — JSON с данными из пройденных режимов. AI использует их для контекста:
- «В прошлом упражнении мы выяснили, что [X]. Давай...»
- Стартовые replies адаптируются под известный контекст
- Не переспрашивает то, что уже известно

## 11. Принципы лендинга (Л1-Л7)

Выведены из 4 книг. Каждый принцип — повторяющийся паттерн, который работает.

| # | Принцип | Суть | Пример |
|---|---------|------|--------|
| Л1 | Hero продаёт результат, не инструмент | Заголовок = что пользователь получит, не что делает AI | ✅ «Перестань играть — начни жить» ❌ «AI-чат по книге Берна» |
| Л2 | Pain cards = голос ЦА | Боли от первого лица, цитаты, не описание проблемы | ✅ «Я столько делаю — а он не ценит» ❌ «Проблема непонимания в отношениях» |
| Л3 | Comparison = 3 альтернативы + AI как мост | Книга (не работает одна), профессионал (дорого/редко), группа/тренинг (высокий барьер), AI (мост) | Профессионал адаптируется: психолог / ТА-терапевт / тренер НЛП / семейный психолог |
| Л4 | Social proof = цифры методологии, не платформы | Тиражи, годы методологии, количество упражнений — не «100 пользователей» | ✅ «15M+ копий продано» ❌ «200 пользователей» |
| Л5 | Anonymous prompt = короткий и провокативный | Демо-чат на ленде — зацепить за 3 сообщения, не «расскажу всё о книге» | Макс 3 обмена до auth wall |
| Л6 | Outcomes привязаны к режимам | Каждый outcome = конкретный режим, не абстрактное обещание | «Распознаёт игры» → режим ta_game_quiz |
| Л7 | Обложка с Литрес, фото автора из открытых источников | cover_url всегда cdn.litres.ru, photo_url — Wikipedia/офф.сайт | Проверить что ссылки работают перед записью |

### Антипаттерны лендинга (НИКОГДА):
- Абстрактные обещания без привязки к функции («Измени свою жизнь!»)
- Стоковые фото или сгенерированные изображения вместо реальной обложки
- Comparison без highlight на AI-тренажёре
- Hero_title длиннее ~50 символов (включая HTML-теги)
- Pain cards с описанием проблемы от третьего лица
- Social proof с маленькими числами платформы (пока нет массы)
- Демо-чат с системным промптом который даёт стену текста в первом сообщении

## 12. Когда книге нужен тест

### Критерии (нужно ≥3 из 5)

| # | Критерий | Пример: да | Пример: нет |
|---|----------|-----------|-------------|
| 1 | Книга содержит диагностическую модель с ≥3 измеримыми шкалами | ISSP: 7 шкал, ТА: 4 эго-состояния | «Атомные привычки»: нет шкал |
| 2 | Шкалы ортогональны (измеряют разное) | Границы ≠ Потребности ≠ Контроль | «Общительность» и «Экстраверсия» — одно и то же |
| 3 | Результат теста персонализирует дальнейший путь (маппинг шкала → режим) | Высокая «Забота» → режим «Границы» | Все идут в один режим |
| 4 | Вопросы формулируются как утверждения с Ликерт-шкалой | «Я часто ставлю потребности других выше своих» (1-5) | Открытые вопросы, эссе |
| 5 | Тест работает как воронка — привлекает новых пользователей | «Узнай свой индекс» — виральный | Тест без shareability |

### Если «да» — принципы дизайна вопросов

**Количество:** 4-7 шкал × 4-7 вопросов = **18-50 вопросов**. Оптимум ~25-35. Меньше 18 — мало для достоверности. Больше 50 — отвал.

**Баланс direct/reverse:**
- Каждая шкала должна иметь и direct, и reverse вопросы (минимум 1 reverse на шкалу)
- Reverse вопросы ловят невнимательные ответы и снижают acquiescence bias
- Reverse = утверждение, где «полностью согласен» = НИЗКИЙ балл по шкале
- Пример direct: «Я часто подавляю свои чувства» → высокий = проблема
- Пример reverse: «Я легко выражаю свои эмоции» → высокий = здорово

**Формулировки:**
- Утверждение от первого лица: «Я...», «Мне...», «Когда...»
- Конкретные поведения, не абстракции: «Я извиняюсь даже когда не виноват» (не «Я слишком добрый»)
- Без двойных отрицаний: ❌ «Я не могу не помогать» → ✅ «Я помогаю даже в ущерб себе»
- Без составных утверждений: ❌ «Я злюсь и грущу одновременно» → ✅ два отдельных вопроса
- Без слов-абсолютов: ❌ «Я ВСЕГДА...» / «Я НИКОГДА...» → ✅ «Мне часто...» / «Я обычно...»

**Шкала ответов (quick_answer_labels):**
- 5-балльная Ликерт по умолчанию: «Совсем нет» — «Скорее нет» — «Иногда» — «Скорее да» — «Полностью»
- Все лейблы одинаковой длины визуально (±2 символа)
- Нейтральный средний вариант (не «Затрудняюсь», а «Иногда»)

**score_direction:**
- `"lower_is_better"` — для тестов синдромов/патологий (ISSP: низкий = здоровый). Высокий балл = проблема
- `"higher_is_better"` — для тестов навыков/ресурсов. Высокий балл = хорошо

**Auth wall:**
- Если тест — точка входа (виральный, на лендинге): auth wall после 60-80% вопросов (мотивация «я уже столько прошёл»)
- Если тест — внутренний инструмент: auth wall = null (пользователь уже залогинен)
- Формула: `auth_wall_question = floor(total_questions * 0.7) - 1` (0-based index)

**Interpretation prompt:**
Промпт для AI-интерпретации должен содержать:
1. Роль («Ты — психолог, специалист по [тема книги]»)
2. Контекст книги (ключевые концепции, шкалы, что они измеряют)
3. Формат ответа (JSON-схема: overall, level_label, scales[], top_zones[])
4. Правила: не диагноз, а самопознание; конструктивный тон; конкретные рекомендации
5. Ссылки на режимы: top_zones[].action_text должен вести к конкретному режиму

**Маппинг шкала → режим:**
Каждая шкала теста должна быть связана с ≥1 режимом через `scales[].exercises`. Это позволяет:
- В результатах теста: «Твоя высокая шкала X → попробуй режим Y»
- В режимах: инжектить данные теста для персонализации

### Если «нет» — альтернативы

| Вместо теста | Когда | Пример |
|-------------|-------|--------|
| Чат-диагностика | Шкалы есть, но вопросы лучше работают в диалоге | Берн: ta_diagnostic (ТА-диагностика через беседу) |
| Без диагностики | Книга не диагностическая | Бакиров: практические навыки, нечего измерять |
| Квиз на лендинге | Нужна воронка, но без полноценного теста | 3-5 вопросов → результат → CTA |

## Что сработало

1. **Анализ ×3 (себя / отношений / родителей)** — три грани, которые покрывают ВСЮ книгу. Каждый анализ имеет чёткий фокус: «Мой синдром» = откуда, «Мои отношения» = как проявляется, «Мои родители» = корни.

2. **«Мои родители и я» вместо «Мой отец и я»** — изначально был только отец. По фидбэку добавили мать как равную тему. В книге мать — вторая половина уравнения (отец ушёл → мать заполнила вакуум). Разделять нельзя — одно без другого неполно.

3. **Экзамен с кейсами из книги** — персонажи Гловера (Алан, Джейсон, Хосе, Тодд, Кал, Шейн) дают готовый материал для кейсов. AI генерирует новые по аналогии. Адаптивная сложность через metacognitive calibration.

4. **Ролевая с двойной ролью** — AI играет антагониста В симуляции и наставника В дебрифинге. Строго разделены маркером 💡. Паттерн из Берна — подтверждён работающим.

5. **Интеграция ISSP → режимы** — маппинг 7 шкал на режимы позволяет персонализировать с первого сообщения. Тест перестаёт быть изолированным — становится навигатором.

6. **Маппинг Темы → Режимы** — существующие Темы (7 шт, привязаны к ISSP-шкалам) теперь ведут в конкретные режимы, а не только в свободный чат. Это подружит старую сущность (темы) с новой (режимы).

## Что пришлось изобретать

1. **Интеграция с существующими Темами** — скилл не описывает кейс, когда у книги уже есть тест и темы в UI. Пришлось добавить секцию маппинга ISSP → режимы и Темы → режимы. → **TODO: добавить в скилл паттерн «интеграция с существующим тестом»**

2. **Переименование режима по фидбэку** — «Мой отец и я» → «Мои родители и я». Скилл предполагает фиксацию на этапе 2, но на этапе 3 фидбэк выявил неполноту. → **Вывод: паузы между этапами критичны, откат дёшев**

## Что заняло больше всего времени

1. Чтение книги на двух языках и извлечение концепций (~20 мин)
2. Проектирование 9 режимов и обоснование выбора (~15 мин)
3. Написание 9 системных промптов (~30 мин)
4. Итерации по фидбэку (родители, ISSP-интеграция, темы) (~15 мин)

## Специфика книги

- **Gender-specific (мужчины)** — все формулировки заточены под мужскую ЦА
- **46 пронумерованных упражнений** — уникальная структура, идеально для воркшопа
- **ISSP тест уже существует** — 7 шкал, инфраструктура, auth wall. Нужна интеграция, не пересоздание
- **Темы уже существуют в UI** — 7 тем привязаны к ISSP-шкалам, нужен маппинг на новые режимы
- **Два типа Nice Guy** — «I'm so bad» и «I'm so good» — AI должен адаптировать тон
- **Персонажи книги** — Алан, Джейсон, Хосе и др. дают готовый материал для кейсов в экзамене

## Уроки для следующих книг

1. Если у книги уже есть тест/диагностика — проектируй интеграцию на этапе 2, а не постфактум
2. Темы (program_themes) — существующая UI-сущность. Новые режимы должны учитывать их маппинг
3. «Родители» > «Отец» в терапевтических книгах — мать почти всегда часть уравнения
4. Паузы между этапами окупаются — фидбэк на этапе 3 дешевле переделки на этапе 4
5. Кейсы из книги = готовый материал для экзамена. Если в книге есть персонажи с историями — используй

## Уроки из аудита 2026-04-18 (book-audit v1.4)

1. **`config.quick_replies` устарел.** Историческое поле — первые страницы (`syndrome`, `relationships`, `parents`, `boundaries`) уже мигрированы на `welcome_replies` с fallback. Страницы `quiz` и `theory` остались на legacy и не показывали кнопки. Любая новая страница для отдельного режима должна читать `welcome_replies` (формат `[{text, type}]`), а не `config.quick_replies` (формат `string[]`).
2. **Дубль `sort_order` визуально не виден** при `enabled=false`-режиме на одной из позиций (был случай self_work на 2 + ng_relationships на 2). При следующей правке порядка карточек — пересчитать всю программу разом, не точечно.
3. **`appendTestScores` теперь сам инжектит правило «не зачитывай числа»** — больше не дублировать его в каждый промпт режима. Старые промпты с этим правилом — ОК (избыточность не вредит), но новые писать без него.

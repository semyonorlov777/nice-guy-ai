# Конспект решений: Игры, в которые играют люди (Берн)

**Slug:** `games-people-play`
**Тип:** Терапия-отношения
**Объединение:** Да — 2 книги → 1 программа
**Дата:** Март 2026 (вторая книга платформы)

## Книги

1. «Игры, в которые играют люди» (Games People Play, 1964) — эго-состояния, транзакции, каталог 36 игр
2. «Люди, которые играют в игры» (What Do You Say After You Say Hello?, 1972) — жизненные сценарии, предписания, разрешение

## Решение по объединению

**Почему объединили:** Первая книга помогает УВИДЕТЬ игры, но не объясняет ПОЧЕМУ человек в них играет. Без второй книги пользователь застревает на «вижу, но не могу остановиться». Вторая книга даёт корни (сценарий) и выход (разрешение).

**Три части программы:**
- **Увидеть** (из книги 1): эго-состояния, транзакции, каталог игр
- **Понять почему** (из книги 2): сценарий, предписания, матрица
- **Что делать** (из книги 2): выход из игры, разрешение, автономия

## Что сработало

1. **Три части как прогрессия** — «Увидеть → Понять → Действовать» создаёт ясный путь. Пользователь не теряется в 10 режимах — видит структуру.

2. **Диагностика с 4 шкалами** — self_worth × trust_others → жизненная позиция (четырёхклеточная матрица). Плюс game_patterns и script_power. Компактнее ISSP (18 vs 35 вопросов) но информативно.

3. **Replies к каждому вопросу теста** — не generic «1-5», а голос пользователя. Делает тест более вовлекающим.

4. **Кросс-режимные связки** — формализованный формат (ПЕРЕДАЁТ → / ПОЛУЧАЕТ ←) помогает проектировать промпты. Режим «Разрешение» получает данные из 3 предыдущих режимов — синтез.

5. **Артефакты** — сценарная матрица (текстовая), формула игры (жирным), разрешение (жирным). Конкретные «вещи», которые остаются после сессии.

6. **Ролевая с двойной ролью AI** — в «Выход из игры» AI играет антагониста И даёт обратную связь, но СТРОГО разделяет (маркер 💡). Работает.

## Специфика книги

- **Объединение двух книг** — уникально для этой программы
- **Каталог 36 игр** — огромный справочный материал для кейсов в «Какие игры ты видишь?»
- **Сценарная матрица** — визуальный артефакт (схема из 4 ячеек), не у каждой книги такое
- **Жизненные позиции** — четырёхклеточная модель, идеально ложится на тест (2 шкалы → 4 комбинации)
- **Gender-neutral** — книга не gender-specific, в отличие от Гловера

## Уроки для следующих книг

1. Если у автора есть 2+ книги, которые дополняют друг друга — объединяй. «Теория + практика» в одной программе > две отдельных
2. Структура «Увидеть → Понять → Действовать» — универсальная прогрессия, подходит для многих книг
3. Формализованные кросс-связки (ПЕРЕДАЁТ →) — помогают не терять данные между режимами
4. Тест нужен каждой книге — определять формат (Ликерт / custom replies / анкета) на этапе 2
5. Таблица технического маппинга на этапе 4 — ускоряет реализацию

## Уроки из аудита 2026-04-18 (после первого UI-прогона теста)

Тест gpp-test был спроектирован, INSERT в `test_configs` сделан — но три обязательных шага из чек-листа PLATFORM_MAP пропущены, и тест в продакшене существовал, но был **полностью невидим и нефункционален**:

1. **`programs.test_system_prompt = NULL`** — AI text-answer streaming уходил с пустым system prompt → текстовые ответы интерпретировались без контекста ТА. Симптом: тест проходится через quick-кнопки, но при текстовом ответе AI не понимает что это тест и отвечает «общими словами».

2. **Не создан `mode_template` + `program_mode` для теста** — карточка теста не появилась в InstrumentList на хабе. Войти в тест можно было только напрямую по URL `/program/games-people-play/test/gpp-test`. Если бы пользователь не знал URL — тест мёртв.

3. **`ta_diagnostic` отсутствовал в `toolKeyMap`** (компонент `components/hub/InstrumentList.tsx`) — chat-based режим с `route_suffix='/chat'` без записи в map → href fallback на `/chat` (свободный чат). Главная free-точка входа GPP вела в неправильный режим. После рефакторинга в auto-derivation проблема снята для всех будущих режимов.

**Вывод:** все три шага PLATFORM_MAP блока «5. mode_template + program_mode для теста» — **не опциональны**. INSERT в `test_configs` сам по себе ничего не даёт. Перед закрытием задачи на тест нужно пройти UI-проверку: «вижу карточку на хабе → клик → welcome → вопрос → результат».
